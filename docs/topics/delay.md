# 查看服务器卡顿情况

正常情况下你应该能有丝滑的游戏体验。进入服务器后，你会在屏幕上方看到一个 Bossbar 并伴随着三个指标，这是 [TabTPS](https://github.com/jpenilla/TabTPS) 插件提供的一项功能。

![TabTPS游戏内显示](./assets/tabtps.jpg)

## 切换指标展示

你可以选择关闭这些指标或者切换其展示的形式。各个形式的预览请参考 [TabTPS 代码库中提供的截图](https://github.com/jpenilla/TabTPS#features)。

- `tab` 在 Tab 菜单（玩家列表）中展示
- `actionbar` 在经验条上方展示
- `bossbar` 在顶端 Bossbar 展示

然后，你可以通过 `/tabtps toggle <形式>` 来切换（执行一次是打开，再执行一次就是关闭）。

## 直接查询指标

你也可以直接使用 `/tickinfo` 或者 `/mspt` 指令查询当前的运行指标，使用 `/ping` 查询你的延迟。

## 三个指标

三个指标的含义如下。

:::tip[TLDR]
简单来说：
- TPS 越高越好，低于 20 说明服务器有卡顿，越低越卡。
- MSPT 越低越好，高于 50 说明服务器有卡顿，越高越卡。低于 50 说明服务器目前十分流畅。
- Ping 越低越好，高于 100 将影响游戏体验。Ping 一般只跟你所处的网络环境有关，而与服务器的运行情况无关。
:::

**TPS** 全称 Tick Per Second（刻每秒），越高越好。20 是它的最大值同时也是正常值，低于 20 就说明服务器出现了卡顿现象。

**MSPT** 全称 Millisecond Per Tick（毫秒每刻），可以简单理解为更换了单位的 TPS 的倒数（但不完全是）。该指标能更细致地反映服务器的运行流畅度。它越低越好。

需要注意的是，MSPT 可以低于其理论最低值 50，因为它表示的并不是目前走过一刻的形式时间，而是目前走过一刻的实际运行时间。如果低于 50，可以说服务器当前的性能出现了盈余。

**Ping** 也就是我们常说的延迟，单位为毫秒，最低值可能为 0（表示几乎没有延迟）。它可以简单理解为从你的电脑到服务器之间的传播时延。这是一个客户端指标，意味着通常情况下，它的数值只与你的网络环境有关，而与服务器的运行情况无关。延迟通常受当前网络质量、与服务器之间的地理距离等因素的影响。我们认为低于 100ms 的延迟是可玩的（playable）。

:::tip[关于 Ping 的参考信息]
截止这篇文章更新的目前，服务器在地理上位于**广东省深圳市**。如果你与深圳的距离较远，可能观察到较高的 Ping 初值（也就是无论如何不会低于的那个值）。

不用担心，只要数字没有上 100，或者你没有直观感受到卡顿，那就是不卡。

另外，即使数字高于 100，大概率也不是地理距离所导致的。
:::

## 反馈

如果你感觉有卡顿，请打开指标展示或者用命令调出指标，并截图，在 QQ 群或者反馈问卷中告诉我，以便我在后台排查。

:::warning[建议及时反馈卡顿]
反馈卡顿时尽量不要等待，因为卡顿的产生原因非常多，且该原因可能只会出现一小段时间（过一会儿可能就消失了）。

如果无法及时反馈，可以先记下卡顿发生的大致时间，这样有助于后续对日志信息的排查。
:::
